## Método del Caso

### Qué es el Método del Caso
    -> Metodología de aprendizaje centrada en el estudiante
    -> Se entrega unos días antes un caso real de una empresa
    -> Se plantea una situación/problema para resolver
    -> La resolución es paradójica: alguien tiene que decidir
    -> Cada uno define qué haría en la posición de ese decisor

### Características del Caso
    -> Plantea información útil para la toma de decisión
    -> También información "basura" de relleno que puede no aportar
    -> Se llega a clase con el caso leído
    -> Se discute entre todos
    -> No lleva nota generalmente
    -> Para aplicar conceptos vistos o por ver
    -> No hay respuestas incorrectas
    -> Los casos son reales pero se cambian nombres de personajes y empresas
    -> Longitud: 2 a 4 carillas máximo

# Modelos de Negocios

## ¿Qué es un Modelo de Negocios?
    -> No es solo vender algo
    -> Tiene que ver con crear, entregar y capturar valor

### Crear Valor
    -> Identificar un problema o necesidad
    -> Ofrecer una solución que ataque esa problemática
    -> Diseñar una propuesta que resuelva cierta necesidad
    -> Ejemplo panadería: pan fresco todas las mañanas
    -> Ejemplo Spotify: resolver necesidad de escuchar música
        -> Sin tener que descargar archivos
        -> Disponible desde cualquier dispositivo
        -> En cualquier momento
        -> En tiempo real

### Entregar Valor
    -> Cómo llega la propuesta al usuario
    -> Cómo hacer que el usuario entienda y consuma la propuesta
    -> Llevar la solución de forma accesible y usable
    -> Ejemplo panadería: local accesible, atención rápida
    -> Ejemplo Spotify: aplicaciones sencillas e intuitivas de usar
        -> Celular, tablet, computadora

### Capturar Valor
    -> Cómo la empresa convierte ese valor en beneficio propio
    -> Obtener ganancia o beneficio a partir de la propuesta de valor
    -> Ejemplo Spotify:
        -> Cobrando suscripciones premium
        -> Vendiendo publicidad
    -> Ejemplo panadería: cobrando por cada pan

## Modelos de Negocio en Software

### Formas de Ganar Plata en Empresas de Software
    -> Suscripciones (mensuales o anuales)
    -> Modelo freemium
        -> No cobro nada por un set de funcionalidades
        -> Si querés otro set, te cobro suscripción
    -> Publicidad
    -> Compra perpetua
        -> Compro la aplicación y ya la tengo
    -> Micro compras / micro transacciones
        -> Juegos con compras dentro del juego
    -> Comisiones por intermediación
    -> Datos (indirectamente, como WhatsApp)
    -> Diversificación de líneas de negocio
        -> Ejemplo MercadoLibre: Mercado Pago, Mercado Envío, crossline

### Definición de Negocio de Software
    -> Se basa en la creación, distribución y comercialización de soluciones digitales
    -> Soluciones digitales: aplicaciones, plataformas, servicios tecnológicos
    -> Aportan valor a los clientes
    -> Productos que no son tangibles en muchos casos
    -> El modelo de negocio define cómo se monetiza

## Ejemplo Práctico: Pedidos Ya

### Actividad Dinámica
    -> Trabajo en pares
    -> Discutir tres preguntas sobre Pedidos Ya:
        -> ¿Quién paga?
        -> ¿Cómo gana dinero la empresa?
        -> ¿Qué valor le da al usuario?

### ¿Quién Paga?
    -> El usuario que pide
    -> Envío
    -> Suscripción por mes (si aplica)
    -> Restaurante paga comisión por cada venta realizada a través de la plataforma

### ¿Cómo Gana Dinero la Empresa?
    -> Comisiones de restaurantes
    -> Publicidad en la web
    -> Posicionamiento en la web (anuncios, promociones para estar más arriba)
    -> Suscripción mensual (te ahorras envíos, descuentos)
    -> Bolsas
    -> Supermercados propios (línea de negocio market)
    -> Diversificación: distintas líneas de negocio
        -> Doctorales, envíos, market
        -> Unidades de negocio

### ¿Qué Valor le da al Usuario?
    -> Comodidad: puede comprar sin moverse
    -> Ahorro de tiempo
    -> Variedad: muchos restaurantes en una misma aplicación
    -> Mayor alcance del mercado para restaurantes
    -> Canal adicional para restaurantes para vender productos
    -> Trabajo para repartidores
    -> Facilita a quienes no podrían sobrevivir de otra forma

## Mercado Objetivo en Software

### Definición
    -> Ejemplo panadería: gente del barrio (mercado local y acotado)
    -> Mercado objetivo puede variar según el producto
    -> En software depende de la aplicación o tecnología

### Características del Mercado de Software
    -> Software puede ser usado por millones de personas en todo el mundo
    -> Sin necesidad de abrir nuevos locales
    -> Solo a través de internet se puede apuntar a lo global
    -> Hoy casi todo depende de tecnología
        -> Comunicación, conexión, servicios, estudio, pagos
    -> Organizaciones tienen dependencia muy marcada de la tecnología
    -> No existe proceso a nivel de organización que no esté soportado por tecnología

### Ejemplos de Necesidades Tecnológicas
    -> Área comercial: CRM (Customer Relationship Management)
        -> Maneja clientes, oportunidades
        -> Guía a los vendedores en el proceso de venta
    -> Área de recursos humanos: software específico
        -> Gestión de colaboradores
        -> Sueldos
        -> Pagos
    -> Área de compras: requiere tecnología
    -> Todas las áreas requieren tecnología

### Observación Importante
    -> Ya no solo las organizaciones grandes y con más plata consumen tecnología
    -> Hoy no hay una sola organización que no consuma tecnología
    -> El mercado objetivo del mundo del software es muy amplio
    -> Todos consumimos tecnología de una u otra forma

## Garner e IDC

### ¿Qué son?
    -> Consultoras internacionales
    -> Analizan todos los rubros de TI
    -> Ejemplo: rubro de antivirus

### El Cuadrante Mágico de Garner
    -> Analiza todos los fabricantes conocidos de un rubro
    -> Los posiciona según:
        -> Visión del producto
        -> Capacidad de ejecución
    -> Cuanto más a la derecha y más arriba, más líderes son
    -> Informe que dice ventajas y desventajas de cada uno
    -> Las empresas líderes compran el informe y lo publican como publicidad
    -> Cobran por acceder al informe

### Forrester Wave
    -> Similar a Garner
    -> Cuanto más arriba y más a la derecha, más líder
    -> El puntito indica porción de mercado
    -> Cuanto más grande el puntito, más porción de mercado

### Otras Referencias
    -> Benchmarks específicos por industria
    -> Ejemplo: LMS (Learning Management Systems)
    -> Comparativos para integración de inteligencia artificial

## Producto vs Servicio

### Definición de Producto
    -> Algo tangible
    -> Lo vendés ya está hecho
    -> Se puede tocar
    -> Es tangible

### Definición de Servicio
    -> Intangible
    -> No se puede tocar
    -> Actividad o proceso que se realiza para el cliente
    -> Relación continua en el tiempo
    -> Puede variar el ancho de banda según la demanda

### En el Mundo del Software

#### Ejemplos Tangibles
    -> Celular: producto
    -> Aplicación descargada: puede ser producto

#### Ejemplos de Servicio
    -> Netflix: producto vendido como servicio
    -> Suscripción mensual para acceder al contenido

#### Evolución: El Caso del Office
    -> Tradicionalmente: comprabas la cajita, llegaba el producto
    -> Migración al modelo en la nube
    -> Dispones el mismo software en la nube
    -> Pagas por el servicio, accedes al mismo software
    -> Tienes actualizaciones en el momento
    -> Transición de licencia perpetua a modelo de suscripción

### Diferencia Clave Producto-Servicio en Software
    -> Servicio de desarrollo de software: servicio
        -> Software factory
        -> Consultoría
    -> Soporte post venta: servicio
    -> Mantenimiento: servicio
    -> Customización específica: servicio
    -> Integración específica: servicio
    -> Producto: transacción en el momento
        -> Me pagas y yo te doy la caja
        -> Es tuyo, lo usas como querés

### Implicaciones Financieras

#### Licencia Perpetua
    -> Se ve como inversión
    -> Desembolso fuerte inicial
    -> Ejemplo: $200,000 por tener la solución

#### Suscripción
    -> Se ve como costo de operar
    -> Pago mensual
    -> Ejemplo: $150 por mes
    -> Permite acceso sin inversión inicial grande
    -> Ventaja: flexibilidad para cortar si no se puede mantener
    -> Permite acceder a productos que antes solo grandes organizaciones podían tener

### Ventajas del Modelo de Suscripción
    -> No hay que desembolsar el monto completo inicialmente
    -> Si a mitad de camino el software no sirve, se puede cambiar
    -> Para empresas chicas o con presupuestos chicos, es posibilidad de acceso
    -> Para empresas nuevas, seguridad de poder cortar el servicio si no pueden mantenerlo
    -> No te comprometes a largo plazo

### Observación
    -> Más que nada en organizaciones, el modelo de suscripción ha ganado terreno
    -> Transición más temprana o más tarde en todos los fabricantes
    -> En el medio había opción de ambos caminos
    -> Hoy casi todo es por suscripción
